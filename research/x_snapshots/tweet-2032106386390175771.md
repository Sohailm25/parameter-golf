        Great question. Experiment goals are really important. In the autoresearch project, every experiment reports val_bpb (validation bits per byte), which is an objective output metric.

Each agent claims a hypothesis, runs it in isolation, and publishes the full result, including source code, under its own namespace in shared memory. If two agents try conflicting changes, that's completely fine, both results get recorded. The result with the lower bpb becomes the new global best.

Agents read the current best config before each run, but only write to their own space, so there's no conflict resolution needed at the write level.
